# Google Replacement Model

AelitaOS aims to reduce Google-centered phone behavior by replacing the default user experience with local-first Aelita components.

## Dependency stance

- No mandatory GMS dependency.
- No cloud assistant requirement.
- No external AI API requirement.

## Optional compatibility path

Future compatibility can be documented in stages:

- no Google path;
- microG-compatible path later;
- user-installed app stores later;
- F-Droid/Aurora-style app acquisition may be documented later, but is not implemented now.

## Experience replacement

Aelita Shell and Local Mind should replace the feeling of Google Discover or Google Assistant with a local, user-owned command center.

- Cloud assistant -> local intent parser and future local model.
- Google-centric defaults -> Aelita-controlled defaults.

## Aelita-controlled defaults

Future ROM work may define Aelita as the default for:

- launcher;
- assistant role;
- search entry;
- app suggestions;
- memory;
- notification summary;
- device state dashboard.

## Current scope

No implementation is included in this PR. This is a planning document only.
